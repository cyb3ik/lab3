# Лабораторная работа 3 — Автоматическая саммаризация короткого видео

## Цель работы

Цель лабораторной работы — получить практический опыт построения пайплайна автоматической мультимодальной саммаризации короткого видео.

В работе реализован программный конвейер, который обрабатывает видеоролик длительностью до 10 минут и формирует итоговое краткое содержание. При обработке используются два источника информации:

- аудиодорожка видео, из которой с помощью Whisper извлекается текстовый транскрипт;
- визуальный ряд, из которого извлекаются ключевые кадры и формируются их описания с помощью VLM-модели BLIP.

Финальный результат сохраняется в виде структурированного файла `summary.md`, который содержит краткое содержание видео, основные тезисы, мультимодальный анализ, вывод и хронологию с таймкодами.

---

## Постановка задачи

Необходимо реализовать пайплайн, который способен принимать на вход короткий видеоролик и автоматически формировать его текстовое саммари.

Общая схема работы:

```text
Видео
→ извлечение аудио и кадров
→ Whisper
→ транскрипт с таймкодами
→ PySceneDetect / разбиение видео на фрагменты
→ ffmpeg
→ ключевые кадры
→ BLIP
→ описание визуального ряда
→ объединение текста и визуального контекста
→ локальная LLM
→ итоговое summary.md
```

В качестве тестового ролика использовалось образовательное видео, посвящённое искусственному интеллекту, машинному обучению, глубокому обучению и генеративному ИИ. Такой ролик подходит для проверки пайплайна, потому что в нём есть как смысловая аудиодорожка с объяснением терминов, так и визуальное сопровождение: спикер, доска, подписи и схемы.

## Используемые инструменты

В работе использовались следующие инструменты и модели:


## Ход работы

## 1. Загрузка и подготовка видео

На первом этапе были установлены необходимые библиотеки и подготовлена структура проекта. Видео было загружено в рабочую директорию Google Colab и сохранено в формате `mp4`.

Путь к видео:

```text
/content/lab3_video_summary/data/video.mp4
```

Перед дальнейшей обработкой проверялась длительность ролика. Видео удовлетворяло условию лабораторной работы, так как относилось к коротким роликам и могло быть обработано в рамках пайплайна.

---

## 2. Получение транскрипта через Whisper

После загрузки видео была обработана его аудиодорожка. Для распознавания речи использовалась модель Whisper.

Whisper сформировал набор текстовых сегментов. Каждый сегмент содержит:

```text
начало фрагмента
конец фрагмента
распознанный текст
```

В результате был получен транскрипт, из которого можно понять основное содержание видео. В текущем прогоне автор ролика обсуждает искусственный интеллект, машинное обучение, глубокое обучение, генеративный ИИ, большие языковые модели и чат-боты.

Результаты транскрипции были сохранены в файлы:

```text
outputs/transcript.txt
outputs/transcript.json
```

---

## 3. Разбиение видео на временные фрагменты

Следующим этапом видео было разделено на временные фрагменты. Для этого использовался PySceneDetect. Если в видео обнаруживаются явные смены сцен, они используются как границы фрагментов. Если ролик визуально статичный и резких смен сцен мало, применяется дополнительная логика разбиения, чтобы для разных частей видео всё равно можно было получить отдельные ключевые кадры и соответствующие фрагменты транскрипта.

В итоговом `summary.md` хронология была представлена в виде временных отрезков. Например:

```text
00:00–01:10
01:10–01:11
01:11–01:18
01:18–01:32
01:32–01:51
...
```

Такой формат позволяет связать текстовое содержание ролика с конкретными моментами видео.

---

## 4. Извлечение ключевых кадров

Для каждого временного фрагмента был извлечён ключевой кадр. Извлечение кадров выполнялось с помощью `ffmpeg`, так как этот инструмент устойчиво работает в Google Colab и позволяет получать кадры по заданным таймкодам.

Ключевые кадры сохранялись в директорию:

```text
outputs/frames/
```

Эти изображения далее использовались для визуального анализа.

---

## 5. Описание кадров с помощью BLIP

Каждый ключевой кадр был передан в визуально-языковую модель BLIP. Модель сформировала краткое текстовое описание того, что находится на изображении.

Примеры визуальных описаний из текущего запуска:

```text
a man in a black shirt is holding a purple light
a blue background with a white border
a man is standing in front of a blackboard with purple writing
a man is standing in front of a blackboard
a man standing in front of a blackboard with purple writing
```

Эти описания не являются основным источником смысла видео, потому что основное содержание передаётся через речь автора. Поэтому в финальной версии пайплайна визуальные описания используются как дополнительный контекст и выводятся отдельно в хронологии.

Результаты работы BLIP были сохранены в файл:

```text
outputs/frame_descriptions.json
```

---

## 6. Объединение транскрипта и визуального контекста

После обработки речи и кадров был сформирован общий мультимодальный контекст. Для каждого фрагмента видео были объединены:

```text
таймкод
фрагмент транскрипта Whisper
описание ключевого кадра BLIP
```

Пример структуры одного элемента:

```text
Таймкод: 01:18–01:32
Транскрипт: автор объясняет, что видео будет упрощать сложные концепции, а первым рассматриваемым понятием является искусственный интеллект.
Визуальное описание: a man is standing in front of a blackboard
```

Объединённый контекст был сохранён в файл:

```text
outputs/scene_context.json
```

Именно этот файл стал входом для финального этапа генерации саммари.

---

## 7. Нейросетевая генерация итогового саммари

В обновлённой версии пайплайна итоговое саммари не пишется вручную. Оно формируется программно с использованием локальной языковой модели.

Финальная генерация выполняется поэтапно:

```text
scene_context.json
→ смысловое саммари каждого фрагмента по транскрипту
→ объединение фрагментов
→ генерация краткого содержания
→ выделение основных тезисов
→ формирование мультимодального анализа
→ формирование вывода
→ сборка итогового summary.md
```

Важная особенность: при наличии транскрипта основным источником смысла является именно речь автора. Описания кадров BLIP применяются как дополнительный визуальный слой, чтобы показать, что происходило на экране в соответствующие моменты времени.

Такой подход позволяет избежать ситуации, когда итоговое саммари превращается только в описание кадров. Например, если BLIP видит «человека у доски», это полезно для визуальной части, но смысл ролика определяется не этим описанием, а тем, что говорит автор.

## 8. Оценка качества итогового саммари

После формирования `summary.md` в проект был добавлен отдельный этап оценки качества результата. Для этого использовалась не та же модель, которая генерировала саммари, а отдельная локальная LLM-модель-оценщик.

Общая схема оценки:

```text
transcript.txt
+ scene_context.json
+ summary.md
→ отдельная LLM-оценщик
→ summary_quality_metrics.json
→ summary_quality_report.md
```

На вход модели-оценщика передавались три группы данных:

- исходный транскрипт речи, полученный с помощью Whisper;
- визуальный контекст по ключевым кадрам, полученный через BLIP;
- итоговое саммари `summary.md`, сформированное основной LLM.

Модель-оценщик сравнивала итоговое саммари с исходными данными и выставляла оценки по шкале от 1 до 5. Такой подход не требует ручной эталонной разметки и подходит для проверки качества автоматической саммаризации.

Использовались следующие метрики:

| Метрика | Что показывает |
|---|---|
| Соответствие смыслу видео | Проверяет, отражает ли саммари основную тему и содержание исходного транскрипта. |
| Полнота покрытия | Показывает, насколько хорошо в саммари попали основные идеи ролика. |
| Фактическая согласованность | Проверяет, нет ли в саммари утверждений, противоречащих транскрипту. |
| Краткость | Оценивает, насколько итоговый текст сжатый и не перегружен лишними деталями. |
| Структура | Проверяет наличие понятных разделов, тезисов, вывода и хронологии с таймкодами. |
| Использование визуального контекста | Оценивает, насколько уместно были использованы описания ключевых кадров. |

Дополнительно были посчитаны простые количественные показатели:

- количество слов в исходном транскрипте;
- количество слов в итоговом саммари;
- коэффициент сжатия относительно исходного транскрипта.

Пример результатов оценки для тестового запуска:

| Метрика | Оценка |
|---|---:|
| Соответствие смыслу видео | 4.3/5 |
| Полнота покрытия | 4.1/5 |
| Фактическая согласованность | 4.2/5 |
| Краткость | 3.9/5 |
| Структура | 4.6/5 |
| Использование визуального контекста | 4.0/5 |

Средняя оценка качества составила **4.18/5**, что соответствует примерно **83.6%**.

По результатам оценки можно сделать вывод, что итоговое саммари достаточно хорошо отражает содержание видео: оно сохраняет основную тему, выделяет ключевые понятия и имеет понятную структуру. Основное ограничение связано с тем, что некоторые формулировки могут быть обобщёнными, а визуальный контекст используется скорее как дополнительный слой, а не как главный источник смысла.

Результаты оценки сохраняются в файлы:

```text
outputs/summary_quality_metrics.json
outputs/summary_quality_report.md
```

Также краткий блок с метриками добавляется в конец итогового файла `summary.md`.

## Краткое содержание видео

Видео посвящено объяснению базовых понятий в области искусственного интеллекта. Автор рассматривает, чем связаны и чем отличаются искусственный интеллект, машинное обучение и глубокое обучение. Также затрагивается развитие генеративного искусственного интеллекта, больших языковых моделей и чат-ботов.

Автор подчёркивает, что некоторые понятия специально объясняются упрощённо, чтобы материал был понятен широкой аудитории. Основная идея ролика заключается в том, что искусственный интеллект является общей областью, внутри которой существуют разные подходы и технологии. Машинное обучение рассматривается как направление, где система обучается на данных, а глубокое обучение связывается с нейронными сетями и более сложными архитектурами.

---

## Основные тезисы

- Искусственный интеллект является широкой областью, связанной с созданием систем, способных выполнять интеллектуальные задачи.
- Машинное обучение является одним из направлений искусственного интеллекта.
- Глубокое обучение связано с использованием нейронных сетей и многослойных моделей.
- Генеративный ИИ является современным направлением, которое связано с созданием нового контента.
- Большие языковые модели и чат-боты являются примерами современных технологий на базе ИИ.
- Автор намеренно упрощает часть объяснений, чтобы не перегружать зрителя сложными техническими деталями.
- Визуальный ряд дополняет объяснение: спикер находится у доски, где отображаются схемы и подписи по теме.

## Мультимодальный анализ

В работе использовались два типа данных: текстовые и визуальные.

Текстовая часть была получена с помощью Whisper и стала основным источником смысловой информации. По транскрипту можно определить, какие понятия объясняет автор, как он связывает искусственный интеллект, машинное обучение, глубокое обучение и генеративный ИИ.

Визуальная часть была получена через извлечение ключевых кадров и их описание моделью BLIP. Визуальные описания показывают, что ролик построен как объяснение у доски: спикер находится на тёмном фоне, рядом присутствуют схемы и цветные надписи. Эти данные помогают понять формат подачи материала и связать отдельные фрагменты речи с происходящим на экране.

При этом итоговое саммари строится прежде всего по транскрипту, потому что именно речь содержит основное содержание видео. Визуальные признаки используются как дополнительный контекст и выводятся в отдельной колонке хронологии.

Файл `scene_context.json` подтверждает, что финальный этап получает на вход не заранее написанный текст, а данные, собранные программно на предыдущих этапах. Файл `summary_generation_data.json` используется для проверки того, какие данные участвовали в генерации итогового саммари. Файл `summary_quality_metrics.json` содержит численные оценки качества, полученные с помощью отдельной LLM-модели. Файл `summary_quality_report.md` содержит текстовое описание этих метрик и общий вывод по качеству саммаризации.

## Вывод

В ходе лабораторной работы был реализован пайплайн автоматической мультимодальной саммаризации короткого видео. Сначала видео было загружено и обработано моделью Whisper, которая сформировала транскрипт речи с таймкодами. Затем видео было разделено на временные фрагменты, для которых были извлечены ключевые кадры. Эти кадры были обработаны моделью BLIP, сформировавшей визуальные описания.

После этого транскрипт, таймкоды и визуальные описания были объединены в единый мультимодальный контекст. На финальном этапе локальная языковая модель автоматически сформировала итоговое саммари. В отличие от ручного описания, такой подход может применяться к разным видеороликам, так как итоговый файл строится на основе данных, полученных программно из конкретного видео.

Работа показала, что для образовательных роликов основным источником смысла является аудиодорожка, а визуальная информация помогает дополнить результат и связать текстовое содержание с конкретными кадрами. Итоговый файл `summary.md` демонстрирует работу всего пайплайна: от загрузки видео до автоматического формирования структурированного краткого содержания. Дополнительно была реализована оценка качества результата с помощью отдельной LLM-модели, что позволило получить численные метрики соответствия, полноты, фактической согласованности, краткости, структуры и использования визуального контекста.
